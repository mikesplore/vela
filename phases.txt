# Remote PC Agent — Coding Agent Master Prompt

## Project Overview

You are building **RemotePC Agent**: a secure, production-grade Python daemon that runs on a Linux PC and exposes a comprehensive REST API. A mobile app (or any HTTP client) can call this API to remotely monitor and control the PC over a local network or secure tunnel (e.g. Tailscale, Cloudflare, or ngrok).

**Stack:**
- Language: Python 3.11+
- API Framework: FastAPI + Uvicorn
- Auth: JWT (Bearer tokens) with a configurable secret
- System integration: `subprocess`, `psutil`, `pydbus`, `pynput`, `Pillow`, `python-xlib`, `dbus-python`
- Packaging: runs as a `systemd` user service
- Config: single `config.yaml` file

---

## Ground Rules for Every Phase

1. **Always write production-quality code** — no stubs, no placeholders, no `# TODO` unless explicitly told to defer something.
2. **Every endpoint must have** proper HTTP status codes, typed request/response Pydantic models, and a docstring.
3. **Error handling is mandatory** — every system call must be wrapped in try/except and return a structured `{"success": false, "error": "..."}` response on failure. Never let an uncaught exception crash the server.
4. **Security is non-negotiable** — every endpoint (except `/health` and `/auth/token`) must require a valid JWT Bearer token. Validate on every request via a FastAPI dependency.
5. **Cross-cutting concerns first** — do not implement features before the project skeleton, auth, and middleware are in place.
6. **File structure must be modular** — one Python module per feature group (see structure below), all routers registered in `main.py`.
7. **Each phase ends with** a `requirements.txt` update listing every new dependency introduced, and a short `CHANGELOG.md` entry.
8. **Test file per module** — write a `tests/test_<module>.py` using `pytest` and `httpx` for every router. Mock all system calls so tests run without root or a real desktop.
9. **No hardcoded values** — secrets, ports, allowed IPs, and paths come from `config.yaml` or environment variables.
10. **All subprocess calls** must use `subprocess.run([...], capture_output=True, text=True, timeout=10)` — never `shell=True` unless absolutely unavoidable, and if so, sanitize all inputs.

### PHASE 1 — Project Skeleton, Config, Auth & Middleware

**Goal:** A running FastAPI server with authentication, logging, and security middleware. No feature endpoints yet.

**Deliverables:**

1. **`config.yaml`** with fields:
   - `host`: bind address (default `0.0.0.0`)
   - `port`: default `8765`
   - `secret_key`: JWT signing secret (warn if default is used)
   - `token_expire_minutes`: default `1440` (24h)
   - `allowed_origins`: list of allowed CORS origins
   - `allowed_ips`: optional IP allowlist (empty = allow all)
   - `feature_flags`: dict of `module_name: true/false` to enable/disable routers
   - `log_level`: default `INFO`

2. **`config.py`** — Pydantic `BaseSettings` class that loads `config.yaml` and allows env var overrides.

3. **`auth.py`** — Implements:
   - `POST /auth/token` — accepts `{ "username": ..., "password": ... }`, validates against a bcrypt-hashed password stored in `config.yaml`, returns a signed JWT access token
   - `verify_token(token)` — dependency function that decodes and validates the JWT, raises HTTP 401 on failure
   - Token payload includes: `sub` (username), `exp`, `iat`

4. **`middleware.py`** — Implements:
   - **Request logger**: logs method, path, status code, and response time for every request
   - **IP allowlist**: if `allowed_ips` is configured, reject all requests not from those IPs with HTTP 403

5. **`dependencies.py`** — Shared dependency `get_current_user` that extracts and verifies JWT from the `Authorization: Bearer <token>` header.

6. **`main.py`** — FastAPI app that:
   - Loads config
   - Registers all middleware
   - Mounts all routers (even if they have no routes yet)
   - Has `GET /health` (no auth) returning `{ "status": "ok", "uptime_seconds": ... }`
   - Has `GET /` (no auth) returning API name, version, and list of enabled modules
   - Starts with Uvicorn via `if __name__ == "__main__"`

7. **`remotepc.service`** — A systemd user-level unit file that:
   - Runs the agent on login
   - Restarts on failure
   - Logs to journald

8. **`tests/test_auth.py`** — Tests for token issuance, invalid credentials, expired tokens, and protected route rejection without token.

### PHASE 8 — Polish, Hardening & Documentation

**Goal:** Make it production-deployable and usable by a mobile client.

**Deliverables:**

1. **Rate limiting** — integrate `slowapi` to rate-limit endpoints. Configurable per-route limits in `config.yaml`.

2. **HTTPS support** — document how to run behind an Nginx reverse proxy with a self-signed cert, OR use `uvicorn` with `--ssl-certfile` / `--ssl-keyfile`.

3. **Auto-generated API docs** — ensure all endpoints have complete OpenAPI descriptions, examples, and response schemas. Accessible at `/docs` (disable in production config if desired).

4. **CLI setup script** (`setup.sh`) that:
   - Installs all `requirements.txt` dependencies
   - Prompts for username and password, hashes it, writes to `config.yaml`
   - Generates a random `secret_key`
   - Installs and enables the `systemd` user service
   - Prints the local IP and access URL

5. **Connection test endpoint** `GET /ping` (no auth) — returns `{ "pong": true }` for quick reachability checks.

6. **`README.md`** covering:
   - Prerequisites (Python version, Linux distro targets, required system packages)
   - Installation via `setup.sh`
   - Manual configuration reference for `config.yaml`
   - How to connect your phone (local network vs Tailscale)
   - Security recommendations
   - How to add a new feature module

7. **Full test run** — all tests must pass with `pytest --tb=short -q`. Aim for >80% coverage.

8. **Final review** — check every endpoint against the security checklist:
   - JWT verified ✓
   - Input sanitized ✓
   - Path traversal prevented ✓
   - Subprocess inputs never interpolated unsanitized ✓
   - Sensitive data never logged ✓
